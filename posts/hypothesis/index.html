Here‚Äôs something problematic: let‚Äôs say you run a hypothesis test at a significance level of Œ±=0.05\alpha=0.05. Then, assuming you ran the test correctly and met assumptions, the probability of a type I error is only 5%. This is the multiple testing problem, and this blog post discusses a couple of solutions to it as well as why they work. Most of this is taken from my notes for UChicago‚Äôs STAT 278, which I‚Äôd highly recommend to anyone interested in these sorts of things.

The set up for the rest of this post is as follows. We have p-values p1,‚ãØ,pnp_1,\cdots, p_n that we‚Äôve acquired from some hypothesis tests. Some of these come from true null hypotheses, and are distributed uniformly on [0,1][0,1]. The rest come from false null hypotheses, and we know nothing about their distribution.

We want to run a test that will have a significance level of Œ±\alpha, not for each test individually, but for all of them together. This will turn out to mean different things for different methods.

1. Bonferroni Correction

This is a method for testing the global null hypothesis, which is the hypothesis that all of the null hypotheses that generated our p-values are true (so we take the intersection of the relevant events). We do this in the most naive way possible, and it turns out to work.

We just test each of the pip_i at a significance level of Œ±/n\alpha/n, and reject the global null if any of the pip_i are rejected. That is, we reject the global null if pi‚â§Œ±/np_i\leq \alpha/n for some ii and fail to reject otherwise.

In what sense does this work?

So that‚Äôs nice. But it‚Äôs also extremely conservative, and we can afford to do things that are a bit more powerful.

Here we‚Äôre also testing the global null, but a bit more cleverly. 3. Holm-Bonferroni

This test is different from the previous two in that it does not test the global null, but instead individually tests each of the pip_i and then controls the family-wise error rate (FWER), which is the probability of making at least one type I error across all the p-values. As an added bonus, we can accept/reject individual p-values rather than accepting/rejecting them all at once, so we have a much better sense of how significant the results are, taken together.

The idea of Holm-Bonferroni is to run Bonferroni until acceptance. We first run Bonferroni on p1,‚ãØ,pnp_1,\cdots, p_n. If we do not reject the global null, we stop. If we do, then we reject the smallest p-value, and repeat this procedure with the other p-values, continuing until we do not reject the global null.
