1. The Characters

Information theory, put crudely, studies a handful of useful quantities. In this section, we introduce those quantities and offer some motivation for them. Throughout, we consider a random variable XX taking values in a finite set UU, with density pp so that ‚Ñô(X=x)=p(x)\mathbb{P}(X=x)=p(x) for any x‚ààUx\in U.

1.1. Entropy (and Friends)

We start with the central quantity of information theory, entropy. Note that we write log\log for the base 2 logarithm, not the natural logarithm.

Definition 1. The entropy of a random variable XX is H(X)=‚àëx‚ààUp(x)log1p(x).H(X)=\sum_{x\in U} p(x)\log \frac{1}{p(x)}.

This expression is pretty weird, but it turns out that it captures ‚Äúthe amount of randomness‚Äù in XX. For some intuition (and sanity checking), we turn to a special case.

Example 2. Consider the case when XX takes two values with probabilities pp and 1‚àíp1-p. Then H(X)=plog1p+(1‚àíp)log11‚àíp.H(X)=p\log\frac{1}{p}+(1-p)\log\frac{1}{1-p}. Some easy calculations show that this function is increasing for p<0.5p<0.5, decreasing for p>0.5p>0.5, maximized at p=0.5p=0.5, and 0 at p=0,1p=0, 1. If we imagine XX as the result of a biased coin, this is the intuitively correct behavior for H(X)H(X): the coin is not random at all when p=0,1p=0, 1 (since you know the outcome before flipping it), and is maximally random when p=0.5p=0.5 (since the outcome is equally likely to be either).

The properties of this example generalize to the case of multiple outcomes.

Proposition 3. The entropy H(X)H(X) satisfies 0‚â§H(X)‚â§log|U|.0\leq H(X)\leq \log |U|. Equality is attained on the left when XX takes a single value and on the right when XX is uniform.

Of course, there are plenty of other functions with these properties, and some of them have more straightforward interpretations e.g.¬†the variance. Why this expression for H(X)H(X)? The idea is to imagine that you see the outcome of XX and have to communicate it to another person using a pre-agreed binary code (so each outcome of XX is transmitted as some binary string). If you design the code so that the outcome xx is encoded with log1p(x)\log \frac{1}{p(x)} bits, then the average number of bits you will need is H(X)H(X). Unfortunately, log1p(x)\log \frac{1}{p(x)} may not be an integer, so you have to use ‚åàlog1p(x)‚åâ\lceil \log \frac{1}{p(x)}\rceil instead. This code is called the Shannon code, and has two nice properties.

Proposition 4. For a random variable XX:

  1. The Shannon code takes at most H(X)+1H(X)+1 bits on average to communicate XX
  2. Any code takes at least H(X)H(X) bits on average

So the Shannon code is in some sense the best possible code, and thus the entropy of XX is a measure of how hard it is to communicate XX. This plays nicely with the idea that entropy captures randomness, since more random quantities are harder to communicate.

1.2. Mutual Information

1.3. KL-Divergence

The last character in our story is the KL-divergence, which is a measure of distance between distributions, but not really.

Definition 8. The KL-divergence between distributions PP and QQ with densities pp and qq respectively, is D(P‚à•Q)=‚àëxp(x)logp(x)q(x).D(P\| Q)=\sum_x p(x) \log \frac{p(x)}{q(x)}.

This sum makes a little more sense written as ‚àëxp(x)log1q(x)‚àí‚àëxp(x)log1p(x).\sum_x p(x)\log \frac{1}{q(x)}-\sum_x p(x)\log \frac{1}{p(x)}. Recalling our discussion of entropy in terms of Shannon codes, the second term is the expected length of a message based on the Shannon code. The first term though, is the expected length of a message if we designed a Shannon code based on QQ instead of PP. The KL-divergence is the difference of these, which is the number of extra bits we use for want of perfect knowledge about the distribution.

Unfortunately, the KL-divergence is not symmetric in P,QP, Q and is not a metric i.e.¬†it does not satisfy the triangle inequality. It does have some nice properties though.
